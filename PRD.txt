1. EXECUTIVE SUMMARY
================================================================================

ABxSPEED is a browser-based network speed testing application that measures
download speed, upload speed, and ping latency. Unlike traditional speed test
tools that require app installs or Java plugins, ABxSPEED runs entirely in
the browser with zero dependencies.

The project aims to provide accurate, adaptive, and visually engaging speed
tests that work on any device — desktop, mobile, or tablet — across WiFi
and mobile data connections.

================================================================================
2. PROBLEM STATEMENT
================================================================================

Existing speed test solutions have limitations:
  - Many require app installations or browser plugins
  - Some impose artificial speed caps on measurements
  - Poor mobile experience with data usage warnings
  - No adaptive testing based on actual connection capability
  - Generic UI that lacks engagement

There is a need for a lightweight, zero-install, browser-native speed test
that adapts to the user's connection and provides accurate results without
artificial limitations.

5. FEATURE REQUIREMENTS
================================================================================

  PRIORITY: MUST HAVE
  ─────────────────────────────────────────────────────────────────────────────

  F-001: Download Speed Test
    - Streams files from Cloudflare edge servers
    - Adaptive sizing: 1MB, 5MB, 10MB, 25MB, 50MB, 100MB
    - Automatically escalates based on measured throughput
    - No speed cap on any tier
    - Uses Fetch API with streaming for accuracy

  F-002: Upload Speed Test
    - Sends data chunks to httpbin.org via POST
    - Adaptive sizing: 1MB, 2MB, 5MB, 10MB
    - Automatically escalates based on measured throughput
    - No speed cap on any tier

  F-003: Ping Test
    - Real HTTP ping (not ICMP)
    - Fetches tiny files (Google favicon) for minimal overhead
    - 10 iterations for statistical accuracy
    - Reports minimum, average, and jitter

  F-004: Connection Type Detection
    - Uses navigator.connection API where available
    - Fallback: latency-based heuristic (WiFi vs Mobile)
    - Fallback: WebRTC bandwidth estimation
    - Fallback: device type detection (mobile/tablet/desktop)
    - Manual selection modal when type is unknown

  F-005: Mobile Data Warning
    - Popup displayed when mobile data is detected
    - Estimates total data usage (~56MB per full test)
    - User must explicitly consent before proceeding
    - Option to run lightweight test only

  F-006: Live Speedometer Gauge
    - Animated SVG gauge with smooth needle movement
    - 6 speed tiers: 10 Mbps, 50 Mbps, 100 Mbps, 500 Mbps, 1 Gbps, 10 Gbps
    - No speed cap — needle freely extends beyond max tier
    - Real-time Mbps display during test
    - Color-coded zones (red → orange → green → cyan → blue → purple)

  F-007: Test Results
    - Score out of 100 for each metric
    - Overall score (weighted average)
    - Speed rating tiers with labels
    - Gbps-aware ratings for fiber connections
    - Comparison with typical speeds

  F-008: Test History
    - Stored in localStorage (client-side only)
    - Shows date, connection type, ping, download, upload
    - Export to JSON
    - Clear history option
    - Maximum 50 entries stored

  PRIORITY: SHOULD HAVE
  ─────────────────────────────────────────────────────────────────────────────

  F-009: Server Selection
    - Auto-select nearest Cloudflare edge
    - Manual override option
    - Multiple test servers for redundancy

  F-010: Share Results
    - Generate shareable link
    - Copy results to clipboard
    - Social media sharing (Twitter, WhatsApp)

  F-011: Dark/Light Theme Toggle
    - Default: dark (cyberpunk)
    - System preference detection
    - Manual toggle with persistence

  PRIORITY: COULD HAVE
  ─────────────────────────────────────────────────────────────────────────────

  F-012: Multi-Language Support
    - English (default)
    - Hindi
    - Spanish

  F-013: PWA Support
    - Service worker for offline access
    - Install prompt
    - App manifest

  F-014: WebSocket Latency Test
    - Real-time bidirectional latency measurement
    - Compare with HTTP ping results

14. APPENDIX
================================================================================

  A. Speed Rating Tiers
  ─────────────────────────────────────────────────────────────────────────────

    Tier        Download       Upload         Ping       Label
    ─────────────────────────────────────────────────────────────────────────────
    Poor        < 5 Mbps       < 1 Mbps       > 100 ms   Slow
    Fair        5-25 Mbps      1-5 Mbps       50-100 ms  Below Average
    Good        25-100 Mbps    5-20 Mbps      20-50 ms   Average
    Great       100-500 Mbps   20-100 Mbps    10-20 ms   Fast
    Excellent   500 Mbps-1G    100-500 Mbps   5-10 ms    Very Fast
    Ultra       > 1 Gbps       > 500 Mbps     < 5 ms     Gigabit+

  B. Adaptive Test Sizes
  ─────────────────────────────────────────────────────────────────────────────

    Tier 1:   1 MB     (Slow connections: < 5 Mbps)
    Tier 2:   5 MB     (Fair connections: 5-25 Mbps)
    Tier 3:  10 MB     (Good connections: 25-100 Mbps)
    Tier 4:  25 MB     (Great connections: 100-500 Mbps)
    Tier 5:  50 MB     (Excellent connections: 500 Mbps - 1 Gbps)
    Tier 6: 100 MB     (Ultra connections: > 1 Gbps)

  C. Scoring Formula
  ─────────────────────────────────────────────────────────────────────────────

    Score = (Ping Score × 0.3) + (Download Score × 0.4) + (Upload Score × 0.3)

    Where each individual score is calculated as:
    - Ping:    100 - (latency_ms / max_latency) × 100
    - Download: min(100, (speed_mbps / target_speed) × 100)
    - Upload:   min(100, (speed_mbps / target_speed) × 100)
